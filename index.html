1. COMPOSITION

Object-Oriented Programming (OOP) Concept

Composition is a design technique where one class contains a reference to another class as a field. This represents a has-a relationship.

Example: A Car has an Engine, so the Car class will have an Engine object inside it.

It is a strong form of association where the lifecycle of the contained object is dependent on the container object.

Composition is preferred over inheritance in many cases because it provides better flexibility and loose coupling.

2. ASSOCIATION

Object-Oriented Programming (OOP) Concept

Association is a general relationship between two classes that are connected logically but have independent lifecycles.

It represents a uses-a or has-a relationship.

There are four types of association: one-to-one, one-to-many, many-to-one, and many-to-many.

Example: A Teacher and a Student are associated with each other — a Teacher can teach many students and vice versa.

Association is the umbrella concept under which aggregation and composition fall.

Association Types:
  • One-to-One: One person has one passport
  • One-to-Many: One teacher teaches many students
  • Many-to-One: Many students have one teacher
  • Many-to-Many: Many teachers teach many students

3. ENCAPSULATION

Object-Oriented Programming (OOP) Concept

Encapsulation is the process of hiding the internal state of an object and restricting access to it by using getter and setter methods.

It allows us to protect the data and expose only what is necessary.

We achieve encapsulation by declaring class variables as private and providing public methods to access or update them.

Encapsulation improves security, maintainability, and flexibility of code.

4. ABSTRACTION

Object-Oriented Programming (OOP) Concept

Abstraction in Java is the process of hiding the implementation details and exposing only the necessary functionalities to the user.

It helps in reducing complexity and increasing code reusability.

We achieve abstraction using abstract classes and interfaces.
Example: In projects, interfaces are used to define service contracts between layers so implementations can vary without affecting consumers.
abstract class Animal { abstract void makeSound(); void eat() { System.out.println("Animal is eating"); } } class Dog extends Animal { void makeSound() { System.out.println("Dog barks"); } } public class Main { public static void main(String[] args) { Animal a = new Dog(); a.makeSound(); a.eat(); } }

5. POLYMORPHISM

Object-Oriented Programming (OOP) Concept

Polymorphism means one name, many forms.

It allows us to use the same method name but perform different behavior depending on the object or context.

Compile-time Polymorphism (Method Overloading)

class Calculator { int add(int a, int b) { return a + b; } double add(double a, double b) { return a + b; } }

Runtime Polymorphism (Method Overriding)

class Animal { void makeSound() { System.out.println("Some sound"); } } class Dog extends Animal { @Override void makeSound() { System.out.println("Bark"); } } public class Main { public static void main(String[] args) { Animal a = new Dog(); a.makeSound(); } }
